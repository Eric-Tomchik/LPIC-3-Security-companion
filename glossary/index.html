ACL
Access Control List — extends Unix DAC with per-user/group file permissions via setfacl / getfacl

ACME
Automatic Certificate Management Environment — protocol used by Let's Encrypt for automated certificate issuance and renewal
Ch. 2
AEAD
Authenticated Encryption with Associated Data — cipher mode providing both confidentiality and integrity (e.g., AES-GCM, ChaCha20-Poly1305)
Ch. 1
AH
Authentication Header — IPsec protocol providing integrity and authentication without encryption
Ch. 14
AIDE
Advanced Intrusion Detection Environment — file integrity checker that detects unauthorized changes by comparing filesystem state to a known baseline
Ch. 7
AppArmor
Path-based Mandatory Access Control system for Linux, using profiles to confine applications. Alternative to SELinux
Ch. 10
Argon2
Memory-hard key derivation function resistant to GPU/ASIC attacks, used by LUKS2 for passphrase hashing. Variants: Argon2i, Argon2d, Argon2id
Ch. 4
ARP
Address Resolution Protocol — maps IPv4 addresses to MAC addresses on local networks
Ch. 15
ASLR
Address Space Layout Randomization — memory protection that randomizes process memory layout, controlled by kernel.randomize_va_space
Ch. 6
AVC
Access Vector Cache — SELinux component that caches access decisions; AVC denial messages in the audit log indicate policy violations
Ch. 10
CA
Certificate Authority — entity that issues and signs digital certificates within a PKI
Ch. 2
CAA
Certification Authority Authorization — DNS record type specifying which CAs are authorized to issue certificates for a domain
Ch. 5
cgroup
Control Group — Linux kernel feature for limiting, isolating, and accounting resource usage of process groups. v2 uses a unified hierarchy
Ch. 8
chroot
Change root — Unix operation that changes the apparent root directory for a process, providing basic filesystem isolation
Ch. 6
conntrack
Connection tracking — netfilter subsystem that tracks the state (NEW, ESTABLISHED, RELATED) of network connections for stateful filtering
Ch. 13
CRL
Certificate Revocation List — a signed list of serial numbers of revoked certificates, published by a CA at regular intervals
Ch. 2
cryptmount
Debian utility for simplified management of encrypted filesystems, allowing non-root users to mount pre-configured LUKS volumes
Ch. 4
CSR
Certificate Signing Request — a message containing the public key and subject information, sent to a CA to request certificate issuance
Ch. 2
CT
Certificate Transparency — public, append-only logs of all issued certificates, designed to detect mis-issuance and rogue CAs
Ch. 2
DAC
Discretionary Access Control — traditional Unix permission model (owner/group/other, SUID/SGID, sticky bit) where owners control access
Ch. 9
DANE
DNS-based Authentication of Named Entities — uses TLSA records to associate TLS certificates with DNS names, bypassing the CA system
Ch. 5
DDoS
Distributed Denial of Service — DoS attack launched from many sources simultaneously to overwhelm the target
Ch. 15
DEP
Data Execution Prevention — hardware/OS feature that prevents execution of code in data memory regions (NX bit)
Ch. 6
DNAT
Destination NAT — iptables/nftables operation that rewrites the destination address/port of packets (port forwarding)
Ch. 13
DNSSEC
DNS Security Extensions — adds cryptographic authentication and integrity to DNS responses using RRSIG, DNSKEY, DS, and NSEC records
Ch. 5
DoH
DNS over HTTPS — encrypts DNS queries using HTTPS on port 443
Ch. 5
DoT
DNS over TLS — encrypts DNS queries using TLS on port 853
Ch. 5
DS
Delegation Signer — DNSSEC record stored in the parent zone containing a hash of the child zone's KSK, establishing the chain of trust
Ch. 5
ECDH
Elliptic Curve Diffie-Hellman — key exchange protocol using elliptic curves, providing equivalent security to DH with smaller keys
Ch. 1
ECDSA
Elliptic Curve Digital Signature Algorithm — signing algorithm using elliptic curves, used in DNSSEC (ECDSAP256SHA256) and TLS
Ch. 1
eCryptfs
Enterprise Cryptographic Filesystem — stacked filesystem that encrypts individual files (not block devices), with per-file FEK/FEFEK key architecture
Ch. 4
ESP
Encapsulating Security Payload — IPsec protocol providing confidentiality, integrity, and authentication; uses protocol number 50
Ch. 14
FEFEK
File Encryption Key Encryption Key — eCryptfs key that encrypts the per-file FEK, derived from the user's mount passphrase
Ch. 4
FEK
File Encryption Key — eCryptfs per-file symmetric key stored in the file header, encrypted by the FEFEK
Ch. 4
HMAC
Hash-based Message Authentication Code — keyed hash function providing message integrity and authentication (used by TSIG)
Ch. 1
IDS
Intrusion Detection System — monitors network or host activity for suspicious behavior and generates alerts
Ch. 12
IKE
Internet Key Exchange — protocol for negotiating IPsec security associations. IKEv2 is the current version, used by strongSwan
Ch. 14
IPS
Intrusion Prevention System — IDS with inline capability to block detected threats in real time
Ch. 12
IP masquerade
Dynamic source NAT using the outgoing interface's IP address — the MASQUERADE target in iptables
Ch. 13
KDF
Key Derivation Function — algorithm that derives cryptographic keys from passwords or other input (PBKDF2, Argon2, scrypt)
Ch. 4
KPTI
Kernel Page Table Isolation — Meltdown mitigation that separates kernel and user-space page tables
Ch. 6
KSK
Key Signing Key — DNSSEC key pair that signs the DNSKEY RRset; the public key's hash forms the DS record in the parent zone
Ch. 5
LSM
Linux Security Modules — kernel framework providing hooks for MAC implementations (SELinux, AppArmor, Smack, TOMOYO)
Ch. 10
LUKS
Linux Unified Key Setup — standard format for disk encryption on Linux, supporting multiple key slots, header metadata, and passphrase management
Ch. 4
MAC
Mandatory Access Control — system-enforced access policy (SELinux, AppArmor, Smack) that overrides user discretion, unlike DAC
Ch. 10
MitM
Man-in-the-Middle — attack where the attacker intercepts and potentially modifies communication between two parties
Ch. 15
MLS
Multi-Level Security — SELinux policy supporting classification levels (e.g., Unclassified, Secret, Top Secret) for government/military use
Ch. 10
mTLS
Mutual TLS — both client and server authenticate with X.509 certificates; enabled with SSLVerifyClient require in Apache
Ch. 3
NAT
Network Address Translation — modifies IP addresses/ports in packet headers during routing; implemented in the iptables nat table
Ch. 13
NBDE
Network-Bound Disk Encryption — auto-unlocks LUKS volumes when a Tang server is reachable on the network, using Clevis as the client
Ch. 4
NDP
Neighbor Discovery Protocol — IPv6 protocol for address resolution, router discovery, and autoconfiguration (replaces ARP)
Ch. 11
NIDS
Network Intrusion Detection System — monitors network traffic for malicious patterns (e.g., Snort, Suricata)
Ch. 12
NSE
Nmap Scripting Engine — Lua-based scripting for extended Nmap functionality; invoked with -sC for default scripts
Ch. 16
NSEC/NSEC3
Next Secure — DNSSEC records providing authenticated denial of existence (proving a domain name does not exist)
Ch. 5
NVT
Network Vulnerability Test — OpenVAS/GVM test scripts written in NASL (Nessus Attack Scripting Language)
Ch. 12
OCSP
Online Certificate Status Protocol — real-time, per-certificate revocation checking via HTTP. OCSP stapling has the server cache responses

pam_tty_audit
PAM module that enables keystroke logging for TTY sessions, recording all user input to the audit log

slice

Ch. 5
tun/tap
Virtual network interfaces: tun operates at Layer 3 (IP packets, routed VPN), tap at Layer 2 (Ethernet frames, bridged VPN)

UEFI

Ch. 6
ulimit
Shell built-in and PAM module (pam_limits.so) for setting per-user resource limits; configured in /etc/security/limits.conf

XSS

ZSK
